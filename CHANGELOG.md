# MoCli 变更日志 (Changelog)

这里记录了 MoCli 项目的重大更新、架构优化以及修复的重点功能。

## [2026-04-17] - 新增对讲机聊天室页面与全局历史记录支撑

### ✨ 新增功能 (Features)

- **可视化文字独立聊天通道**：在设置中心新增了完整的「💬 对话 (Chat)」工作台界面，允许用户抛开语音引擎束缚，采用直接手键键入文字的方式流转测试。新模块拥有独立的文字气泡 UI 及流畅的滚动历史面板。
- **全局会话本地留痕记忆**：底层重构了 `llm` 引擎的保存逻辑模块，使 SQLite `mocli.db` 直接沉淀支持全局所有的 `["role": "user", "role": "assistant"]` 双端历史记录，将每次问答无缝序列化固化，为将来查拉与复盘做好了保障。

---


## [2026-04-15] - 多航点音画同步巡航引擎 & 交互系统大重构

### ✨ 新增功能 (Features)

- **多点连飞巡航 (Multi-Point Cruising)**：彻底颠覆单点往返逻辑！AI 现在的解说可以一气呵成包含多组 `[POINT:x,y:标题]` 坐标。光标能够像真人解说员一样，从一个目标点横空飞向下一个目标点驻留，直到整段解说结束再统一优雅归隐归位。
- **动态气泡双轨刷新**：光标自带的气泡提示框不再死板。上方加粗显示指归元素的【头衔】，下方气泡随着 TTS 发音和句子节点【实时滚动字幕】。做到了“飞哪说哪、说哪显哪”的工业级别演播体验。
- **免流快速测试通道 (Zero-Cost Debugging)**：专门配置了纯内建 Python 软截停指令。只要对麦克风说出“测试”或“调试”，将硬核拦截网络与截屏通讯并免 Token 回复一段包含了 4 处屏幕不同角的长文本极限连飞指令，用于测试音频流。

### 🔄 架构重塑 (Refactoring)

- **调度中枢移交**：撤销了 `main.py` 以往“见坐标便擅自起飞”的武断调度。全面确立以 `tts.py` 为绝对核心节拍器的总线机制。长文统统扔进合成缓冲队列。
- **音频指令融合队列**：`tts.py` 中的 `speaker_queue` 升维打击，现已化身为可以同时入列 `[二进制 PCM 音频包]`、`[坐标起飞事件]`、`[字幕同步事件]` 和 `[哨兵终结者]` 的异构控制队列。
- **静默环境保鲜**：新增兜底仿真延时算法。如果在设置中关掉了 TTS 本地语音合成服务，系统将通过一套基于字符长度的仿生阅读流速（约每四字/秒）主动挂起 `asyncio.sleep`，让所有的 UI 气泡、光标驻留仍能完全按照常理时间停留变换！

### 🐛 修复 (Bug Fixes)

- 根除了大段发音导致的跨线程阻塞死锁。
- 修改并杜绝了 Python 内部作用域冲突（Local Variable Shadowing）导致在快速发令“测试”后走常规查询链遭遇 `capture_screen` 变量空引用的致错 Bug。

## [历史更新汇总]

- **UI 彻底焕新升级**：移除了粗糙的旧版 UI，全面换装为 Windows 11 “Fluent Design” 光滑质感风格大面板。增加 “大模型”、“语音”、“高级参数” 三栏式导航。
- **实时物理雷达**：增加 `settings.py` 内动态刷新实时捕获实际鼠标屏幕指针绝对物理位置像素的 Debug 器，并自带投弹 15 秒打点验证坐标发射试验田。
- **KWS/STT 分级解耦**：允许彻底分别单独关闭麦克风侦听或者唤醒词模型，并增设底气十足的 Push-To-Talk F10 直连快捷通道机制。
- **拼音自动预转**：加入了根据中文唤醒词自动基于 `pypinyin` 分离音素的服务逻辑，降低 Sherpa-ONNX 门槛。
- **Qwen3-TTS 强流式集成**：搭建对接专门打造好的超低延迟 API 音管，实现一呼百应的对话体验。
